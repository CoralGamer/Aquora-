# Fase 3.5: Rediseño Visual, Expansión Open Source y Ajuste de Resiliencia (Fase Intermedia)

Esta fase intermedia actúa como un puente entre la consolidación del portal web administrativo (Fase 3) y el desarrollo de la aplicación móvil (Fase 4). Su propósito es refinar la estética visual de AQUORA utilizando principios de diseño premium inspirados en mockups modernos de alto impacto (estilo *minimalist dark landscape* con acentos rojos finos), expandir significativamente la documentación del Firmware Open Source y garantizar un diseño adaptable en dispositivos móviles y de escritorio.

## 1. Racional del Rediseño Visual (Inspiración Premium)

Tomando como base la estética premium compartida, adaptamos los siguientes elementos dentro del ecosistema web de AQUORA sin modificar la narrativa ni el texto del proyecto:

### A. Paleta de Colores y Contraste Extremo
*   **Fondo Negro Absoluto (`#000000` / `#020617`):** Reemplaza las tarjetas grisáceas tradicionales por páneles de vidrio con un desenfoque de fondo (`backdrop-filter`) extremo (16px a 24px) y fondos oscuros puros. Esto hace que las imágenes y los datos de telemetría "floten" con un contraste de vanguardia.
*   **Líneas de Acento y Umbrales Críticos (`#f43f5e`):** Se introducen líneas horizontales ultrafinas (de 1px) en color rojo escarlata/carmesí. Estas líneas no son solo decorativas: representan el **Umbral de Alerta Sanitaria de SIVIGILA** (casos de EDA históricos).
*   **Contraste Tipográfico de Alto Rango:** Títulos en fuente *Outfit* con peso extra-bold (800) y tamaños de hasta 3.5rem, contrastados con descripciones en *Inter* con colores grisáceos atenuados (`#94a3b8`) para priorizar la legibilidad.

### B. Elementos de Interfaz Incorporados
1.  **Indicadores Verticales de Milestones/Slides:** En la Landing Page se incluye un sistema de numeración vertical (`01`, `02`, `03`, `04` con indicador activo de alto contraste), ideal para estructurar el recorrido de la problemática hídrica en La Guajira.
2.  **Tarjetas de Recomendación e Impacto Geográfico:** Las secciones informativas usan tarjetas con imágenes de fondo a pantalla completa con un gradiente oscuro inferior, mostrando a la comunidad Wayúu y el territorio en un formato cinematográfico.
3.  **Botones con Borde Activo Neón:** Bordes carmesí y cian con resplandores sutiles para guiar al usuario hacia la acción principal (Iniciar Sesión o ver la Documentación).

## 3. Expansión de la Documentación Open Source (Firmware ESP32)

El portal de documentación para la comunidad se ha expandido sustancialmente para garantizar que cualquier líder comunitario o técnico en territorio pueda compilar, cablear e instalar el cerebro del filtro.

### A. Guía de Instalación Paso a Paso (Arduino IDE)
1.  **Descargar e Instalar Arduino IDE:** Descarga la versión 2.x desde el sitio web oficial de Arduino.
2.  **Instalar el Soporte para ESP32:**
    *   Ve a `Archivo` > `Preferencias`.
    *   En "Gestor de URLs Adicionales de Tarjetas", añade: `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`.
    *   Ve a `Herramientas` > `Placa` > `Gestor de Tarjetas`, busca `esp32` e instala la última versión por Espressif Systems.
3.  **Instalar Librerías Requeridas:**
    *   Abre el Gestor de Librerías (`Programa` > `Incluir Librería` > `Gestionar Librerías`).
    *   Busca e instala: **NewPing** (para sensor ultrasónico) y **DallasTemperature** + **OneWire** (si se acopla sensor de temperatura del agua).
4.  **Seleccionar la Placa y Puerto:**
    *   Conecta tu ESP32 por USB.
    *   Selecciona `DOIT ESP32 DEVKIT V1` o `ESP32 Dev Module`.
    *   Elige el puerto COM respectivo (ej. `COM3` en Windows, `/dev/ttyUSB0` en Linux).

### B. Código C++ Oficial Expandido para Telemetría
Este código inyecta directamente lecturas simuladas realistas o lecturas físicas de los sensores de TDS (Sólidos Disueltos), Turbidez y Nivel de agua, transmitiendo por protocolo HTTP POST a la API de FastAPI que a su vez escribe en Supabase.

```cpp
#include <WiFi.h>
#include <HTTPClient.h>

// ==========================================
// CONFIGURACIÓN DE RED Y SEGURIDAD
// ==========================================
const char* ssid     = "Wokwi-GUEST";       // SSID por defecto en Wokwi o red local
const char* password = "";                  // Sin clave para simulación
const char* serverURL = "http://127.0.0.1:8000/api/v1/readings"; // Endpoint seguro

// Clave única asignada por Abaco (Obtenida desde el portal público)
const String deviceKey = "DEV_ESP32_GUAF8392"; 

// Pines de Hardware
#define PIN_TDS A0
#define PIN_TURB A1
#define PIN_TRIG 12
#define PIN_ECHO 13
#define LED_RED 2
#define LED_GREEN 4

void setup() {
  Serial.begin(115200);
  pinMode(LED_RED, OUTPUT);
  pinMode(LED_GREEN, OUTPUT);
  
  // Inicialización de Red
  WiFi.begin(ssid, password);
  Serial.print("Conectando a la red de la comunidad");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
    digitalWrite(LED_RED, !digitalRead(LED_RED)); // Parpadeo lento indicando sin red
  }
  
  Serial.println("\n¡Red enlazada con éxito!");
  digitalWrite(LED_RED, LOW);
  digitalWrite(LED_GREEN, HIGH); // Luz verde fija indica conexión activa
}

void loop() {
  if (WiFi.status() == WL_CONNECTED) {
    // 1. Lectura y cálculo de sensores (Filtro físico)
    int rawTds = analogRead(PIN_TDS);
    float voltageTds = rawTds * (3.3 / 4095.0);
    // Ecuación de calibración TDS (Simulada en Wokwi)
    float tdsValue = (133.3 * voltageTds * voltageTds * voltageTds - 255.86 * voltageTds * voltageTds + 857.39 * voltageTds) * 0.5;

    int rawTurb = analogRead(PIN_TURB);
    float voltageTurb = rawTurb * (3.3 / 4095.0);
    // Ecuación de conversión de Voltaje a NTU
    float turbidityValue = -1120.4 * voltageTurb * voltageTurb + 5742.3 * voltageTurb - 4352.9;
    if (turbidityValue < 0) turbidityValue = 0.0;

    // Medición del Tanque (HC-SR04)
    digitalWrite(PIN_TRIG, LOW);
    delayMicroseconds(2);
    digitalWrite(PIN_TRIG, HIGH);
    delayMicroseconds(10);
    digitalWrite(PIN_TRIG, LOW);
    long duration = pulseIn(PIN_ECHO, HIGH);
    float distanceCm = duration * 0.034 / 2;
    float levelPercent = 100.0 - ((distanceCm / 100.0) * 100.0); // Asumiendo tanque de 1 metro
    if (levelPercent < 0) levelPercent = 0.0;
    if (levelPercent > 100) levelPercent = 100.0;

    // 2. Transmisión HTTP Segura
    HTTPClient http;
    http.begin(serverURL);
    http.addHeader("Content-Type", "application/json");

    // Construcción del Payload JSON libre de inyecciones
    String jsonPayload = "{\"device_key\":\"" + deviceKey + 
                         "\",\"tds_ppm\":" + String(tdsValue, 2) + 
                         ",\"turbidity_ntu\":" + String(turbidityValue, 2) + 
                         ",\"water_level_pct\":" + String(levelPercent, 2) + "}";

    Serial.print("Transmitiendo datos a la nube... ");
    int httpResponseCode = http.POST(jsonPayload);

    if (httpResponseCode > 0) {
      String response = http.getString();
      Serial.println("Código HTTP: " + String(httpResponseCode));
      Serial.println("Respuesta del servidor: " + response);
      
      // Alerta de mantenimiento física
      if (tdsValue > 400.0 || turbidityValue > 5.0) {
        digitalWrite(LED_RED, HIGH); // Alerta local encendida
      } else {
        digitalWrite(LED_RED, LOW);
      }
    } else {
      Serial.print("Error de transmisión: ");
      Serial.println(httpResponseCode);
    }
    http.end();
  } else {
    Serial.println("Se perdió la conexión WiFi. Intentando reconectar...");
    WiFi.begin(ssid, password);
  }
  
  delay(15000); // Transmisión constante cada 15 segundos
}
```

### C. Preguntas Frecuentes (FAQs) sobre la Instalación
*   **¿Qué pasa si la red WiFi es inestable en territorio?**
    El microcontrolador ESP32 incluye lógica de autoreconexión automática. Si se pierde el enlace a la API, los LEDs rojo/verde parpadearán alertando de la falta de datos.
*   **¿Con qué frecuencia debo limpiar las capas físicas del filtro purificador?**
    *   **Bagazo de Caña de Azúcar:** Reemplazo recomendado cada 3 meses para asegurar una adsorción óptima de metales y químicos.
    *   **Zeolita Activa:** Lavado físico con salmuera cada 6 meses.
    *   **Arena Silícea:** Lavado de sedimentos acumulados una vez al mes por retrolavado simple.

## 5. Walkthrough Unificado de Implementación (Fases 3 y 3.5)

El siguiente es el flujo operativo completo verificado y consolidado para garantizar el funcionamiento del ecosistema híbrido en simulaciones locales y entornos de producción.

### Paso 1: Configurar la Base de Datos Relacional (Supabase)
1.  Ejecuta el script SQL [setup_auth.sql](file:///e:/AQUORA/backend/setup_auth.sql) en el Editor SQL de tu consola de Supabase. Esto creará la tabla `user_profiles` y activará el trigger automático en PostgreSQL que replica los nuevos usuarios creados desde la autenticación.
2.  Ejecuta el script de siembra rica `scratch/seed_rich_data.py` utilizando Python para inyectar las 8 comunidades de La Guajira, cargando registros históricos realistas de brotes de EDA del SIVIGILA y asociando dispositivos piloto.

### Paso 2: Iniciar la Puerta de Enlace (FastAPI Backend)
1.  Levanta tu entorno virtual e inicia el servidor Uvicorn en tu terminal del backend:
    ```bash
    uvicorn main:app --reload --host 0.0.0.0 --port 8000
    ```
2.  Si estás realizando pruebas en red local con tu dispositivo móvil o utilizando simuladores en la web como Wokwi, expón tu puerto con un túnel de ngrok:
    ```bash
    ngrok http 8000
    ```

### Paso 3: Inicializar la Simulación de Hardware (Wokwi)
1.  Abre la simulación de Wokwi configurada con el circuito ESP32.
2.  Pega el **Código C++ Oficial** detallado en esta guía en la pestaña de código, asegurándote de reemplazar la constante `serverURL` con tu URL pública de ngrok (o tu dirección IPv4 local) y la `deviceKey` con una clave autorizada.
3.  Inicia la simulación para transmitir datos de TDS, turbidez y nivel de agua cada 15 segundos de forma constante.

### Paso 4: Levantar y Operar el Portal Web Integrado (Vite)
1.  Ingresa al directorio del frontend e inicializa el servidor de desarrollo:
    ```bash
    npm run dev
    ```
2.  Abre el portal en tu navegador. Experimenta el flujo multi-rol:
    *   **Sección Pública:** Explora el Hero visual de La Guajira con acentos carmesí. Abre la Documentación Open Source ampliada para revisar las guías de ensamble. Solicita una nueva `device_key` rellenando el formulario técnico.
    *   **Ingreso Seguro:** Inicia sesión con la cuenta de administrador provista (ej. `abaco.admin@aquora.org`).
    *   **Centro Analítico de Ábaco:** Observa el mapa territorial interactivo de 580px cargando los 8 nodos de La Guajira. Presiona un marcador para evaluar los reportes de EDA correlacionados con la telemetría viva y ver los gráficos dinámicos de Recharts.
    *   **Aprovisionamiento Segurizado:** Registra una nueva cuenta de miembro comunitario asociando su correo, contraseña temporal y el ID de dispositivo inactivo registrado previamente por el portal público.
    *   **Dashboard Familiar:** Cierra sesión e ingresa con la cuenta del nuevo miembro. Visualiza en exclusiva los indicadores dinámicos y umbrales de su filtro asignado, configurando sus preferencias de alerta vía WhatsApp/Correo en la columna JSONB de Supabase.
